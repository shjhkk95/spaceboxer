Space Boxers
Game Design Document

by Brendan Kondracki, Yong chen, SeokHoon Kim

INTRODUCTION

This document will describe the overall view of "Space Boxers", a top-down, 2-D game developed in the javascript programming language. This game will make use of multiple game design techniques such as collision detection, path finding, and artificial intelligence.


TECHNOLOGY

Space Boxers will be developed as a desktop-based web game, created in javascript using the Phaser library and Tiled application.


BACK STORY

For centuries, space boxing has been the most popular sport in the galaxy. Few people on earth know of its existence....until now! For the first time ever, you will get to play as your very own space boxer, taking on harder and more advanced opponents along your journey.


OBJECTIVE

Your task is to master the art of space boxing, level by level. As the game progresses, you will be forced to take on a larger number of opponents, with more obstacles standing between you and victory. Should you be knocked out at any point in your journey, you will fall off the edge of space forever.


GAMEPLAY

Space Boxers will be formatted as a top-down, tile based game. The player will be able to move left, right, up, and down the arena. Since the arena will be smaller than the viewport, no scrolling will be required.

As the user, your goal will be to knock out each of the enemy bots from the arena, into outer space, by approaching them in close proximity, and meleeing them. Enemy bots, however, will have the ability to shoot projectiles at you from any distance. In order to prevent being hit by these projectiles, you will have to switch your block's color to match that of the attacker, and melee the projectile before it makes contact with your body. Throughout the play arena, there will be open holes which either your or the bots can fall through, so caution must be used when moving around the arena. Once all enemy bots have been eliminated, the level will complete, and the user will be able to move onto the next level.


CONTROLS

This game will be played using both a keyboard and mouse. Once started, use the following:

  • A - Move Left
  • D - Move Right
  • W - Move Up
  • S - Move Down
  • LEFT-ARROW - Change Block Color
  • RIGHT-ARROW - Change Block Color (Opposite Direction)
  • SPACE - Melee
  • ESC - Pause/Unpause the game. If a game is not in progress, ESC does nothing.

GRAPHICAL USER INTERFACE

As far as the GUI is concerned, if we consider

  • Splash Screen - The splash screen GUI simply presents a game logo and a Start button for the user to press when they are ready to play.

  • Main Menu - The main menu of the game will be deployed with the following options:
    • Start Game - This option will start the last unlocked level by the user.
    • Select Level - This button will take the player to a level selection screen, where they may choose to play any previously unlocked level.
    • Controls - If pressed, the game will display an info screen that includes a description of all game controls.
    • Help - If pressed, the game will display an info screen with information about the game authors.

  • In-Game GUI - Once a level has been selected, the player will be loaded into the arena, and a countdown timer will show them when to begin playing. Once the timer has completed, all that will be displayed on the screen is the player, enemy bots, the arena, and the surrounding space enviornment.

    • ARTWORK

    All artwork in the game will be original. The following needs to be created:

    • Player/Enemy Bots - We will need to design colored blocks to be used by both the player, and enemy bots. Since the number of possible colors employed by the player will increase as the game goes on, the number of colored sprites we create will need to be proportional to the number of levels that can be reached by the player.

    • Arena - The arena will be a squared area in space, with multiple holes scattered around it. We will need to design tiles to represent the area with which the player can move across.

    • Space Background - In order to make sure the player is aware of the areas in the viewport with which to avoid, we will need to design the space background enviornment.

    SOUND EFFECTS

    All sound effects will be original. Sounds must be made to coincide with each of the following events:

    • Enemy Shooting
    • Player Melee
    • Enemy Being Contacted By Melee
    • Enemy/Player Being Knocked Out Of Arena
    • Player Changing Color
